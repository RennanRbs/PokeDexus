# Histórico de evoluções – Portal Pokémon

Este documento registra cada evolução (entrega) do projeto Portal Pokémon. Cada linha corresponde a uma user story concluída, alinhada ao [backlog do produto](backlog-portal-pokemon.md). Ao implementar uma US, faça o commit no padrão definido em [GIT_WORKFLOW.md](GIT_WORKFLOW.md) e atualize a **Data** desta tabela.

## Legenda

- **Versão:** Numeração semântica por épico (ex.: 1.0.0 = Épico 1 completo; 1.1.0 = Épico 2 completo).
- **Data:** Data da entrega (preencher ao concluir a US); use `-` até implementar.
